# PRD — אפליקציית מנהל משימות (Todo App)

תמצית הפרויקט
- מטרה: לבנות אפליקציית Todo קטנה ומלמדת ב-React 19 + Vite, ב-JavaScript בלבד, עם קומפוננטות פונקציונליות ושימוש ב-Hooks.
- קהל יעד: סטודנטים בלימודי React שמבקשים דוגמה פרקטית שניתן להסביר ולבנות בשלבים.

טכנולוגיה
- React 19
- Vite (ללא שינוי)
- JavaScript (ES6+)
- קומפוננטות פונקציונליות בלבד
- שימוש ב-Hooks: `useState`, `useEffect`
- שמירה ב-`localStorage`
- CSS חופשי (פשוט, אפשרי להשתמש ב-CSS רגיל או מודולים)

דרישות פונקציונליות
- הוספת משימה חדשה עם תיאור טקסטי.
- סימון משימה כהושלמה / ביטול סימון (toggle).
- עריכת טקסט של משימה קיימת.
- מחיקת משימה.
- סינון תצוגת המשימות לפי: הכל / פעילות בלבד / הושלמו בלבד.
- סימון ויזואלי של הפילטר הפעיל (button או tab מודגש).
- הצגת מונה של משימות פעילות (שאינן הושלמו).
- שמירת מצב המשימות ב-`localStorage` אוטומטית בעת שינוי.
- שחזור המשימות מ-`localStorage` בעת טעינת האפליקציה.
- לכל משימה יש מזהה ייחודי (`id`).

דרישות ארכיטקטורה וקומפוננטות
- לפחות 4 קומפוננטות React נפרדות עם הפרדת אחריות:
  - `App` (קומפוננטת שורש): מנהלת את ה-state הראשי של המשימות, אחראית על שמירה/שחזור מ-`localStorage`, מעבירה props ו-callbacks לילדים.
  - `TodoList`: מציגה רשימת משימות לפי המצב/פילטר, מקבלת פעולות (toggle, edit, delete) כ-props.
  - `TodoItem`: קומפוננטה לפריט משימה בודד — מציגה טקסט, checkbox/toggle, כפתורי עריכה ומחיקה.
  - `AddTodo` (או `TodoForm`): טופס להוספת משימה חדשה ולעריכה כאשר נדרש.
  - `FilterBar` (אם נדרש): רכיב נפרד לבחירת פילטרים (All / Active / Completed) והצגת המונה.
- אין לרכז את כל הלוגיקה בקומפוננטה אחת; כל פעולה שתשנה state תועבר דרך callbacks מהשורש.

ניהול state
- ה-state הראשי (`todos`) יישמר בקומפוננטת השורש `App` כ-array של אובייקטים.
- מבנה לכל משימה (דוגמה):
  ```
  {
    id: 'uuid-or-timestamp',
    text: 'תיאור המשימה',
    completed: false,
    createdAt: 1670000000000
  }
  ```
- Props זורמים כלפי מטה; אירועים (callbacks) זורמים כלפי מעלה (לדוגמה: `onAddTodo`, `onToggleTodo`, `onEditTodo`, `onDeleteTodo`).
- אין שימוש בספריות לניהול state חיצוניות (Redux וכו').

שמירה ו-persistence
- מפתח `localStorage` מוצע: `my-react-todo.todos`.
- בעת אתחול היישום (`useEffect` ברמת ה-App) נטען את המצב אם קיים.
- בכל שינוי ב-`todos` נעדכן את ה-`localStorage` (רק לאחר שה-state השתנה).

UI / חוויית משתמש
- ממשק פשוט ונקי המאפשר הוספה/עריכה/סינון/מחיקה במהירות.
- כפתורי הפילטר יהיו גלויים וברורים; הפילטר הפעיל יהיה מודגש ויזואלית.
- בעת עריכה — אפשר inline edit או modal קטן; יש לשמור על פשטות וממשק שקל להסביר.
- הצגת מונה משימות פעילות במקום ברור (למשל בצד ה-FilterBar).

איכות קוד
- שמות משתנים וקומפוננטות משמעותיים וברורים.
- שימוש ב-ES6+ (arrow functions, destructuring, spread/rest וכו').
- קוד קריא ופשוט להסבר — להימנע מ-over-engineering.
- אין שגיאות או אזהרות בקונסול בדפדפן.
- אין שימוש ב-Class Components.

בדיקות וקבלה
- Acceptance Criteria (קריטריונים לקבלה):
  - ניתן להוסיף משימה חדשה ותהיה שמורה ב-`localStorage`.
  - ניתן לסמן משימה כהושלמה ולהסיר סימון.
  - ניתן לערוך טקסט של משימה קיימת.
  - ניתן למחוק משימה.
  - הפילטרים (All/Active/Completed) עובדים כמצופה; הפילטר הפעיל מודגש.
  - המונה מציג את מספר המשימות הפעילות הנכון.
  - טעינת הדף משחזרת את ה-todos מ-`localStorage` בלי שגיאות.

מטרות פיתוח ושלבי עבודה (לסטודנט)
1. לכתוב את ה-`PRD` ולוודא הבנה (קובץ זה).
2. ליישם את `App` עם state בסיסי וטעינת דמה.
3. לבנות את `AddTodo` ולוודא שמוסיפים פריטים ל-state.
4. לבנות `TodoList` ו-`TodoItem` ולממש toggle/delete/edit.
5. לממש את `FilterBar` ואת המונה.
6. להוסיף שמירה/שחזור ב-`localStorage` ולבדוק ללא שגיאות.
7. ללטש CSS ולוודא נגישות בסיסית.

הגבלות ודרישות נוספות
- אסור לשנות את כלי הבנייה (להישאר עם Vite).
- אין להשתמש ב-TypeScript.
- אין להשתמש בספריות חיצוניות לניהול state.
- יש לשמור על קוד שהסטודנט יוכל להסביר בקלות.
