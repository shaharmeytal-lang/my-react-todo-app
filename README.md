EXPLANATION — Todo App

קובץ זה נותן הסבר קצר על ההחלטות והצעדים בפרויקט עבור המורה/בוחן.

1) מה הקוד עושה בקצרה
- אפליקציית Todo קטנה שמאפשרת הוספה, סימון כהושלמה/לא הושלמה, עריכה, מחיקה וסינון (All / Active / Completed).
- נתוני המשימות נשמרים ב-`localStorage` ומטענים בעת אתחול.

2) ארכיטקטורה וקבצים רלוונטיים
- `src/App.jsx` — קומפוננטת שורש; מחזיקה את ה-state הראשי (`todos`) ומממשת פונקציות ל-add/toggle/edit/delete; מטפלת ב-persistence ב-`localStorage`.
- `src/components/AddTodo.jsx` — טופס מבוקר להוספת פריטים.
- `src/components/TodoList.jsx` — מציג את רשימת הפריטים.
- `src/components/TodoItem.jsx` — פריט בודד; תומך ב-toggle, edit inline ו-delete.
- `src/components/FilterBar.jsx` — כפתורי פילטר ומונה משימות פעילות.
- `src/styles/todo.css` — סגנון בסיסי.
- `prd.md` — דרישות המערכת והתנאים שניתנו בתחילת הפרויקט.

3) שיפוטיות והחלטות עיצוביות
- קומפוננטות פונקציונליות בלבד; שימוש ב-Hooks (`useState`, `useEffect`).
- אין שימוש ב-Redux או בניהול state חיצוני; כל ה-state נמצא ב-`App`.
- מזהים ייחודיים נוצרים בעזרת `Date.now()` + חלק אקראי קטן — מספק לפרויקט לימודי.
- שמתי דגש על קריאות הקוד ופשטות כדי שהסטודנט יוכל להסביר כל חלק.

4) הוראות להרצה
```bash
npm install
npm run dev
# פתח בדפדפן: http://localhost:5173/ (או הפורט שמודפס בטרמינל)
```

5) בדיקות מהירות שנעשו
- בדקתי: הוספה, סימון/ביטול, עריכה, מחיקה, סינון, מנגנון שמירה/שחזור ב-localStorage.
- תיקנתי אזהרות נפוצות (מזהה כפול, כפתורי type, aria attributes) כדי להקטין אזהרות בקונסול.
